# Documentation

Welcome to the project documentation.

This documentation is organized by engineering discipline rather than by feature.

## Documentation Structure

| Directory | Purpose |
|-----------|---------|
| adr/ | Architecture Decision Records |
| architecture/ | System architecture and design |
| decision-log/ | Chronological engineering decisions |
| deployment/ | Local and production deployment guides |
| development/ | Development standards and workflows |
| operations/ | Monitoring, maintenance, and operations |
| templates/ | Reusable documentation templates |

## Documentation Principles

- Documentation evolves with the project.
- Every significant architectural decision should have an ADR.
- Deployment procedures must be reproducible.
- Documentation is treated as part of the product.
